# 応答時の注意事項

- 思考は英語で、応答は必ず日本語で。
- いきなり修正をするのではなく、先に修正の方針を端的に示してください。
- パッケージの一次情報を必ず確認すること。
- 自分の一般的な知識で対応しようとしないこと。パッケージは公式の実装がすべてです。それをきちんと確認すること。
- 何が問題かを特定することが、結果的に問題解決に早く辿り着くことを理解すること。
- 手っ取り早く思いついた案やエラーログに対応する即物的な修正は避け、問題を包括的に捉え、そもそも問題は何か？について、深く思考してください。
- 謝るふりをすることが思考ではありません。謝る暇があるなら思考して。
- 必要に応じてコンソール・ログやエラーログを取得して、状況の把握に努めてください。状況の把握をすることなく、経験即で解決法を特定するのは避けてください。
- 勝手な推測によるコードを書くな。
- Netlifyでデプロイする際は、未使用の変数はエラーとなるので避ける。
